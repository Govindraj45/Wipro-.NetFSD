# Problem Statement 42 - Online Book Store REST API

`BookStoreApi` provides:

- `GET /api/books`
- `GET /api/books/{id}`
- `POST /api/books`
- `PUT /api/books/{id}`
- `DELETE /api/books/{id}`
- `GET /api/authors`
- `GET /api/authors/{id}`
- `GET /api/authors/{authorId}/books`
- `POST /api/authors`
- `PUT /api/authors/{id}`
- `DELETE /api/authors/{id}`

The project uses in-memory storage, attribute routing, validation attributes, and test coverage for core API behavior.
