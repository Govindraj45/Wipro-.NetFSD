# Problem Statement 43 - Online Movie Catalog REST API

`MovieCatalogApi` provides:

- `GET /api/movies`
- `GET /api/movies/{id}`
- `POST /api/movies`
- `PUT /api/movies/{id}`
- `DELETE /api/movies/{id}`
- `GET /api/directors`
- `GET /api/directors/{id}`
- `GET /api/directors/{directorId}/movies`
- `POST /api/directors`
- `PUT /api/directors/{id}`
- `DELETE /api/directors/{id}`

The project uses in-memory storage, attribute routing, validation attributes, and test coverage for core API behavior.
